# LSP Setup

SourceKit-LSP is configured via `buildServer.json` (project root, git-ignored) generated by `xcode-build-server`. This bridges the FurAffinity Xcode target and FAKit SPM package into a single index, enabling cross-module `goToDefinition`, `hover`, and `findReferences`.

## Requirements

- `xcode-build-server` installed: `brew install xcode-build-server`
- Project built at least once (populates DerivedData index store)

## Regenerating buildServer.json

Run from the project root whenever DerivedData is wiped or the scheme changes:

```
xcode-build-server config -project FurAffinity.xcodeproj -scheme FurAffinity
```

Then rebuild:

```
xcodebuildmcp simulator build --project-path FurAffinity.xcodeproj --scheme FurAffinity --simulator-name "iPhone 17"
```

## LSP operation coverage

| Operation | Same-module | Cross-module (FurAffinity ↔ FAKit) |
|---|---|---|
| `documentSymbol` | ✅ always | ✅ always |
| `hover` | ✅ after build | ✅ after build + buildServer.json |
| `goToDefinition` | ✅ after build | ✅ after build + buildServer.json |
| `findReferences` | ✅ after build | ✅ after build + buildServer.json |
| `workspaceSymbol` | ⚠️ via script only | ⚠️ via script only |

## workspaceSymbol

sourcekit-lsp **does** fully support `workspace/symbol` (including cross-module
FurAffinity ↔ FAKit) once the index store is populated. The editor/agent `LSP`
tool's `workspaceSymbol` operation is position-based — it takes a file + line +
character but no query string — so it can only ever send sourcekit-lsp an empty
query, and sourcekit-lsp returns nothing for empty queries (and for any query
shorter than 3 characters). This is a limitation of how the tool maps a cursor
position onto a query-based LSP method, **not** a sourcekit-lsp or buildServer
configuration issue; no `buildServer.json` change can fix it.

To search symbols across the workspace, drive sourcekit-lsp directly:

```
Scripts/lsp-workspace-symbol.py <query>            # query must be ≥ 3 chars
Scripts/lsp-workspace-symbol.py FASession --kind interface
Scripts/lsp-workspace-symbol.py OnlineFASession --kind class
```

The script starts sourcekit-lsp, opens a seed file to load the index, then sends
a real `workspace/symbol` query and prints `kind`, name, and `path:line`. It
relies on the same populated index store and `buildServer.json` as the other
operations, so build the project at least once first.
